36 C Program TESTEPH
37 C ---------------
38 C
39 C TESTEPH tests the JPL ephemeris reading and interpolating routine using
40 C examples computed from the original ephemeris.
41 C
42 C TESTEPH contains the reading and interpolating subroutines that are of
43 C eventual interest to the user. Once TESTEPH is working correctly, the
44 C user can extract those subroutines and the installation process is complete.
45 C
46 C You must supply "testpo.XXX" to TESTEPH, via standard input. "testpo.XXX
47 C is the specially formatted text file that contains the test cases for the
48 C ephmeris, DEXXX.
49 C
50 C After the initial identifying text which is concluded by an "EOT" in
51 C columns 1-3, the test file contains the following quantities:
52 C
53 C JPL Ephemeris Number
54 C calendar date
55 C Julian Ephemeris Date
56 C target number (1-Mercury, ...,3-Earth, ,,,9-Pluto, 10-Moon, 11-Sun,
57 C 12-Solar System Barycenter, 13-Earth-Moon Barycenter
58 C 14-Nutations, 15-Librations)
59 C center number (same codes as target number)
60 C coordinate number (1-x, 2-y, ... 6-zdot)
61 C coordinate [au, au/day]
62 C
63 C For each test case input, TESTEPH
64 C
65 C - computes the corresponding state from data contained
66 C in DExxx,
67 C
68 C - compares the two sets,
69 C
70 C - writes an error message if the difference between
71 C any of the state components is greater than 10**(-13).
72 C
73 C - writes state and difference information for every 100th
74 C test case processed.
75 C
76 C This program is written in standard Fortran-77.
77 C
78 C HOWEVER, there are two parts which are compiler dependent; both have
79 C to do with opening and reading a direct-access file. They are dealt
80 C with in the subroutine FSIZERi, i=1,3. (There are three versions of
81 C this subroutine.
82 C
83 C 1) The parameter RECL in the OPEN statement is the number of units per
84 C record. For some compilers, it is given in bytes; in some, it is given
85 C in single precision words. In the subroutine FSIZER of TESTEPH, the
86 C parameter NRECL must be set to 4 if RECL is given in bytes; NRECL must
87 C be set to 1 if RECL is given in words. (If in doubt, use 4 for UNIX;
88 C 1 for VAX and PC)
89 C
90 C 2) Also for the OPEN statement, the program needs to know the exact value
91 C of RECL (number of single precision words times NRECL). Since this
92 C varies from one JPL ephemeris to another, RECL must be determined somehow
93 C and given to the OPEN statement. There are three methods, depending
94 C upon the compiler. We have included three versions of the subroutine
95 C FSIZER, one for each method.
96 C
97 C a) Use the INQUIRE statement to find the length of the records
98 C automatically before opening the file. This works for VAX's;
99 C not in UNIX.
100 C
101 C b) Open the file with an arbitrary value of RECL, read the first record,
102 C and use the information on that record to determine the exact value
103 C of RECL. Then, close the file and re-open it with the exact value.
104 C This seems to work for UNIX compilers as long as the initial value of
105 C RECL is less than the exact value but large enough to get the required
106 C information from the first file. (For other compilers, this doesn't
107 C work since you can open a file only with the exact value of RECL.)
108 C
109 C c) Hardwire the value of RECL. This number is NRECL*1652 for DE200,
110 C NRECL*2036 for DE405, and NRECL*1456 for DE406.
111 C
